# EXPLICACION DEL PROYECTO

## 1. Arquitectura del Proyecto
El proyecto es una aplicación Full-Stack moderna dividida en dos componentes principales:
* Backend (.NET 8 Web API): Un servicio RESTful encargado de exponer endpoints, gestionar los datos en la base de datos MySQL usando Entity Framework Core, autorizar usuarios, y aplicar la lógica de negocio (búsqueda de mesas libres).
* Frontend (React + Vite + Tailwind): Una aplicación SPA (Single Page Application) que consume la API REST. Tailwind CSS se utiliza para ofrecer un diseño visual premium y muy fluido, inspirado en plataformas como TheFork o Resy.

## 2. Flujo de JWT (JSON Web Tokens)
Para manejar la autenticación y que el sistema sea seguro:
1. Registro/Login: El usuario envía sus credenciales al backend (`POST /api/auth/login`). Si son correctas, el backend firma un token JWT usando la clave secreta definida en `appsettings.json` y lo devuelve al cliente.
2. Almacenamiento: El frontend recibe el JWT y lo almacena localmente en `localStorage`.
3. Peticiones Protegidas: Cuando el usuario intenta hacer una reserva, el frontend recupera el JWT y lo envía en el encabezado (Header) de la petición HTTP: `Authorization: Bearer <token>`.
4. Validación Automática: El backend intercepta la petición entrante, verifica matemáticamente la firma del token (para asegurar que nadie lo ha manipulado) y si es válido, lee automáticamente qué usuario es, y registra la información correctamente.

## 3. Configuración de Base de Datos para Hostinger
Para apuntar a tu base de datos y desplegar en Hostinger, debes actualizar las credenciales en el archivo `backend/appsettings.json`.
Debes ubicar la clave `"DefaultConnection"` y modificar exactamente los valores que empiezan con `TU_`:
```json
"ConnectionStrings": { 
  "DefaultConnection": "Server=TU_IP;Database=TU_BD;User=TU_USUARIO;Password=TU_PASS;" 
}
```
Reemplázalos por los parámetros reales provistos en el panel web de Hostinger MySQL Databases.

## 4. Comandos para Levantar el Proyecto

### Para el Backend
Abre una ventana de terminal, dirígete a la carpeta `backend` y ejecuta:
```bash
cd backend
dotnet run
```
La API se habilitará (generalmente en `http://localhost:5000` u o el puerto que muestre la consola).

### Para el Frontend
Abre una nueva ventana de terminal, dirígete a la carpeta `frontend` y ejecuta:
```bash
cd frontend
npm run dev
```
Tu web frontend de React se lanzará con Vite (generalmente en `http://localhost:5173` o el que muestre la consola). Simplemente haz CTRL+Clic (o CMD+Clic) en el enlace que te aparezca.
